# Scripted

AI-generated talk-show videos. Pick a host, drop a topic, get a watchable monologue with synced lip movement, on-demand dubbing, and a chat panel that knows the script.

Powered end-to-end by the Google Gen AI SDK (`@google/genai`): **Gemini 3.5 Flash** for research + script + chat, **Veo 3.1** for video clips, **Imagen 4** for host portraits, and **Gemini 2.5 Flash TTS** for dubbed audio.

## What it does

1. **Pick a host** — three generic monologue templates (Ava, Marcus, Leo), each with a portrait generated by Imagen. Personality is intentionally adaptive — voice emerges from the topic, not a hardcoded style.
2. **Drop a topic** — free text, news URL, or Hacker News link. Gemini researches it deeply.
3. **Generate** — Gemini writes an N-clip script (`durationSeconds / 8` clips); Veo renders each 8s clip in parallel using the supplied reference image so the host stays the same person across every shot; ffmpeg stitches them into one MP4; served from `/public/files` (or uploaded to Mux when configured).
4. **Watch** — Mux-style player, synced transcript, research context, in-page chat with the show ("ask the host"), and on-demand audio dubbing into Spanish, French, German, Portuguese, or Japanese.

## Stack

- **Next.js 16** (App Router, Turbopack) + React 19 + TypeScript
- **Tailwind CSS 4** with a `liquid-glass` dark theme + boomerang video background (gsap parallax)
- **`@google/genai` 2.6** — single SDK for Gemini / Veo / Imagen / TTS
- **PostgreSQL** via Drizzle ORM — templates, shows, clips, chat
- **Vercel Workflow DevKit** — durable, retryable show-generation pipeline
- **`fluent-ffmpeg`** + system `ffmpeg` — clip stitching
- **Mux** (optional) — when configured, uploads stitched MP4s for HLS streaming; falls back to local `/public/files` playback
- **Remotion** (optional) — for social-clip renders
- **ElevenLabs** (optional) — alternate audio translation path

## Quick start

```bash
# 1. deps
npm install

# 2. database (PostgreSQL with pgvector)
npm run db:migrate

# 3. seed the 3 generic host templates + Imagen portraits
npm run seed-templates

# 4. dev server
npm run dev
```

App at `http://localhost:3000`.

## Environment

Create `.env.local`. See `app/lib/env.ts` for the full Zod-validated schema.

### Required

```bash
DATABASE_URL=postgres://...

# Single key OR a comma-separated pool (pool takes precedence)
GOOGLE_GENERATIVE_AI_API_KEY=AIza...
# Multiple keys = round-robin → N× parallel Veo throughput
GOOGLE_GENERATIVE_AI_API_KEYS=AIza-key-1,AIza-key-2,AIza-key-3
```

### HTTP API

| Method | Path                                  | Body / Query                                      |
| ------ | ------------------------------------- | ------------------------------------------------- |
| POST   | `/api/workflows/generate-show`        | `{ showId }` → starts the durable workflow        |
| POST   | `/api/tts`                            | `{ transcript, hosts, targetLang? }` → `audio/wav` |
| POST   | `/api/upload-image`                   | `multipart/form-data` `file` → `{ url }`          |
| POST   | `/api/lambda/render`                  | Triggers Remotion social-clip render              |
| GET    | `/api/lambda/progress`                | Polls Remotion progress                           |
| POST   | `/api/workflows/translate-audio`      | ElevenLabs dub pipeline                           |
| POST   | `/api/workflows/translate-captions`   | Caption translation pipeline                      |

Server actions (not REST, called from React) handle create-show, chat send/get, template CRUD, and status polling. See `app/**/actions.ts`.

## Notes on the Veo path

- **Models:** default `veo-3.1-generate-preview`. Same model for reference image and first/last-frame interpolation.
- **Per-clip cap:** Veo is hard-limited to 8s per generation. Longer videos = N clips stitched.
- **Identity locking:** every monologue clip is sent the template's reference image *plus* an explicit "the SAME individual depicted in the reference image — preserve facial features, hairstyle, age, skin tone, clothing exactly across every shot" instruction, so the host doesn't morph across clips.
- **API key pool:** `GOOGLE_GENERATIVE_AI_API_KEYS` lets each Veo job pin a different key for its start → poll → download lifecycle. With N keys and N parallel clips, throughput is roughly N× a single-key setup.
- **Vertex-only flags:** `personGeneration`, `generateAudio`, `enhancePrompt`, `negativePrompt` are accepted by the SDK but the Gemini Developer API rejects them with 400. They're opt-in per call (`VeoCallOptions`) for Vertex deployments.
